What We Claim — and How to Verify It

Claim 1
Your notebook data never leaves your device.
The notebook stores everything in localStorage — your browser's local storage. No API call sends notebook content anywhere. Verify: Open notebook.html on GitHub and search for fetch — the only network call is for search queries, never for your written entries.
Claim 2
The privacy tools run entirely on your device.
The sanitizer, converter, and leak score tools process files using client-side JavaScript. No file is ever uploaded. Verify: Open any tool, disconnect your WiFi, and use it. It still works. Or read the source: sanitizer, converter, leak score.
Claim 3
GitHub Pages cannot run server-side code or log your visits.
GitHub Pages is a static file host. It serves HTML, CSS, and JS files — nothing else. GitHub does log standard web server access (IP addresses in their infrastructure), but D2D has no access to those logs and cannot add custom tracking. Verify: GitHub Pages documentation confirms it serves static content only.
Claim 4
The API server code matches what's on GitHub.
The server running on Render auto-deploys from github.com/deathtodata/fortune0.com on every push. The /health endpoint reports the running version. Verify: Compare curl https://fortune0-com.onrender.com/health version field against the latest commit on GitHub. Render's deploy logs are tied to specific git commits.
Claim 5
Search queries are logged for functionality, not sold.
The server stores search queries in an activity table to power features like search count on the about page. This data is NOT shared with third parties. The server.py source shows exactly what's stored: email (if logged in), action type, and detail string. Verify: Read the log_activity() function in server.py. There are zero outbound API calls that transmit user data.
Honest Disclosure
What we DO collect.
We are honest about what touches our server: search queries go through our Render API (needed to proxy results), story mode sends URLs to our server (needed for AI analysis), and Stripe handles all payment data (we never see your card number). The activity table on our database logs actions like "search" and "login" with timestamps. We believe this is the minimum needed to run the service. If you disagree, the code is open — tell us what to change.
